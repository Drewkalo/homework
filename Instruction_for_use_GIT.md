# Инструкции для работы с Git через SSH

### Как создать SSH ключ

1.  Откройте терминал.
2.  Введите следующую команду:

    bash
    ssh-keygen -t ed25519 -C "ваша_электронная_почта@example.com"
    ```

    Замените "ваша_электронная_почта@example.com" на ваш адрес электронной почты, привязанный к аккаунту GitHub.

3.  После ввода команды вам будет предложено выбрать место для сохранения ключа.  Можно оставить предложенное по умолчанию (обычно ~/.ssh/id_ed25519)  просто нажав Enter.
4.  Вам будет предложено ввести passphrase (парольную фразу) для дополнительной защиты.  Если хотите использовать passphrase, введите её дважды.  Если не хотите, просто нажмите Enter дважды.
5.  В результате в каталоге ~/.ssh/ будут созданы два файла: id_ed25519 (приватный ключ) и id_ed25519.pub (публичный ключ).  Приватный ключ нельзя никому передавать!

▌Как добавить ключ в аккаунт на GitHub

1.  Откройте публичный ключ id_ed25519.pub в текстовом редакторе.
2.  Скопируйте всё содержимое файла.
3.  Перейдите на сайт GitHub и войдите в свой аккаунт.
4.  Нажмите на свою аватарку в правом верхнем углу, затем выберите "Settings".
5.  В боковом меню выберите "SSH and GPG keys".
6.  Нажмите кнопку "New SSH key" или "Add SSH key".
7.  В поле "Title" введите описание ключа (например, "Домашний компьютер").
8.  В поле "Key" вставьте скопированное содержимое публичного ключа.
9.  Нажмите кнопку "Add SSH key".  Возможно, потребуется ввести пароль от аккаунта GitHub для подтверждения.

▌Как склонировать репозиторий

1.  На сайте GitHub перейдите к репозиторию, который вы хотите склонировать.
2.  Нажмите кнопку "Code" (зеленого цвета).
3.  В появившемся окне выберите вкладку "SSH".
4.  Скопируйте SSH URL репозитория (он будет начинаться с git@github.com:).
5.  Откройте терминал.
6.  Перейдите в каталог, куда вы хотите склонировать репозиторий.
7.  Введите следующую команду:

        git clone git@github.com:ИМЯ_ПОЛЬЗОВАТЕЛЯ/НАЗВАНИЕ_РЕПОЗИТОРИЯ.git
    

    Замените git@github.com:ИМЯ_ПОЛЬЗОВАТЕЛЯ/НАЗВАНИЕ_РЕПОЗИТОРИЯ.git на скопированный SSH URL репозитория.

8.  Если при клонировании появится запрос на подтверждение подлинности хоста, введите yes.
9.  После завершения клонирования в выбранном каталоге появится папка с названием репозитория.
```
